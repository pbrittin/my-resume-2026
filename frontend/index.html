About This Project

This is not a Wix page or a hosted template — it is a fully serverless web application deployed on AWS, built as a hands-on implementation of the AWS Cloud Resume Challenge. The goal is to demonstrate real cloud engineering skills through infrastructure as code, CI/CD automation, serverless architecture, and security best practices — all visible in the public GitHub repository.

Frontend

Amazon S3 — Static File Hosting Amazon CloudFront — Global CDN AWS Certificate Manager — TLS/HTTPS Custom Domain (pmbxdevseceng.com) Vanilla HTML · CSS · JavaScript

Backend — Visitor Counter

Amazon API Gateway — REST API AWS Lambda — Python 3.12 · ARM64 (Graviton2) Amazon DynamoDB — On-Demand Billing

Infrastructure & CI/CD

AWS SAM — Infrastructure as Code AWS CloudFormation GitHub Actions — Automated Pipelines OIDC Federation — Zero Stored Credentials pytest — 90%+ Code Coverage Required

Security

Least-Privilege IAM Roles Private S3 Bucket (CloudFront-only access) GitHub OIDC — No Long-Lived AWS Keys in CI/CD

How It Works

The static site is stored in a private S3 bucket and served globally through CloudFront with HTTPS via an ACM certificate on a custom domain. Every page load triggers a JavaScript fetch() to an API Gateway endpoint. The request invokes a Lambda function (Python 3.12, ARM64/Graviton2) that atomically increments a counter in DynamoDB and returns the updated count — displayed in the Page views badge at the top of this page.

Every push to the main branch triggers two independent GitHub Actions pipelines. The frontend pipeline validates HTML, syncs assets to S3, and invalidates the CloudFront cache. The backend pipeline runs the test suite (minimum 90% coverage enforced), validates the SAM template, and deploys via CloudFormation — all authenticated through OIDC federation with no AWS credentials stored in GitHub.
